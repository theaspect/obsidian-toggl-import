A -- HI-01: Fix NaN-unsafe wrapMinutes in src/api.ts (~line 103)

The current code:
    const wrapMinutes = (wrapH ?? 0) * 60 + (wrapM ?? 0);

Replace with:
    const wrapMinutes = isNaN(wrapH) || isNaN(wrapM) ? 0 : wrapH * 60 + wrapM;

Why: Number("abc") returns NaN. The ?? 0 operator does NOT catch NaN (NaN is not
null/undefined). When wrapMinutes is NaN, the guard wrapMinutes === 0 is false so the
filter branch runs, but startMinutes >= NaN is always false -- every entry is silently
dropped. The isNaN guards make any malformed input degrade to 0 (no-op, no filtering).
