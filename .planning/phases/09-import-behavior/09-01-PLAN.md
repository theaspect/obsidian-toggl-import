<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filename → date parser | Active note basename is user-controlled input; prefix capture is the attack surface |
| Settings persistence → API behavior | sortOrder from data.json controls API result ordering |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | Filename prefix regex in main.ts | mitigate | Regex `/^(\d{4}-\d{2}-\d{2})/` only captures `\d` characters — no injection possible. Captured group is passed as a date string to the Toggl API which handles invalid calendar dates gracefully (returns empty results). No eval or shell execution. |
| T-09-02 | Information Disclosure | sortOrder in data.json | accept | Sort order preference is non-sensitive config (asc/desc). Same risk profile as existing outputFormat field already in data.json. No PII or credentials involved. |
| T-09-03 | Tampering | sortOrder value read from data.json | mitigate | `plugin.settings.sortOrder` is typed `'asc' \| 'desc'`. The ternary `=== 'desc' ? -cmp : cmp` defaults to ascending for any non-'desc' value, so a tampered data.json value (e.g., injected string) produces ascending sort at worst — no code execution. |
| T-09-04 | Denial of Service | Repeated prefix regex match per command invocation | accept | Regex match is O(n) on basename length; basenames are bounded by filesystem limits. No DoS vector. |
</threat_model>
