Helper to configure mockRequestUrl responses based on URL pattern:
```typescript
function setupMockResponses(opts: { entries?: any[]; projects?: any[]; meWorkspaceId?: number } = {}) {
	mockRequestUrl.mockImplementation(async (req: { url: string }) => {
		if (req.url.includes('/me/time_entries')) {
			return { status: 200, json: opts.entries ?? SAMPLE_ENTRIES, headers: {}, text: '', arrayBuffer: new ArrayBuffer(0) };
		}
		if (req.url.includes('/projects')) {
			return { status: 200, json: opts.projects ?? SAMPLE_PROJECTS, headers: {}, text: '', arrayBuffer: new ArrayBuffer(0) };
		}
		if (req.url.endsWith('/me')) {
			return { status: 200, json: { default_workspace_id: opts.meWorkspaceId ?? 42 }, headers: {}, text: '', arrayBuffer: new ArrayBuffer(0) };
		}
		return { status: 404, json: null, headers: {}, text: '', arrayBuffer: new ArrayBuffer(0) };
	});
}
```

**describe('fetchTimeEntries'):**

1. `it('constructs date boundaries with local midnight')` — call fetchTimeEntries(plugin, '2024-06-15'), verify mockRequestUrl was called with a URL containing `start_date=` and `end_date=` params that both include `2024-06-15T` or `2024-06-14T` (depends on timezone offset direction, so just verify the params exist and are ISO format).

2. `it('filters out entries with negative duration (running)')` — include entry with duration: -1 in mock response, verify returned array does not contain it.

3. `it('filters out entries with null stop (running)')` — include entry with stop: null, duration: -1 in mock response, verify returned array does not contain it.

4. `it('preserves start as UTC ISO string for downstream formatting')` — verify returned entry.start === '2024-06-15T09:00:00+00:00' (same as API response).

5. `it('resolves project_name from project cache')` — verify returned entry with project_id 100 has project_name 'Project Alpha'.

6. `it('sets project_name to empty string when project_id is null')` — verify entry with project_id null has project_name === ''.

7. `it('normalizes null description to empty string')` — verify entry with null description in API response has description === '' in returned TimeEntry.

8. `it('normalizes null tags to empty array')` — verify entry with null tags has tags deep-equal [].

9. `it('throws specific error on 401')` — mock requestUrl to return status 401 for time_entries URL, expect fetchTimeEntries to reject with Error message exactly 'Toggl API error: invalid API token (401)'.

10. `it('throws specific error on 429')` — mock status 429, expect message exactly 'Toggl API error: rate limited — try again in a moment (429)'.

11. `it('throws on network failure')` — mock requestUrl to throw (simulating network error), expect message exactly 'Toggl API error: network request failed'.

12. `it('auto-populates workspaceId from /me when workspaceId is 0')` — create plugin with workspaceId: 0, call fetchTimeEntries, verify mockRequestUrl was called with URL ending '/me', and plugin.saveSettings was called.

13. `it('skips /me call when workspaceId is already set')` — create plugin with workspaceId: 42, call fetchTimeEntries, verify no call to URL ending '/me'.

14. `it('does not include API token in error messages')` — mock 401, catch the error, verify error.message does NOT contain 'test-token-abc123'.

Each test calls `beforeEach(() => { mockRequestUrl.mockReset(); })` to reset mocks between tests.
