**Step 3: Implement template evaluator in `src/formatter.ts`**

Add a `renderTemplate` function (not exported — internal to formatter.ts):

```typescript
function renderTemplate(template: string, vars: Record<string, string>): string {
    const KNOWN_VARS = ['description', 'start', 'duration', 'tags', 'project'];
    // Step 1: Temporarily replace unknown ${...} tokens so they survive evaluation
    const placeholders: Map<string, string> = new Map();
    let idx = 0;
    const escaped = template.replace(/\$\{([^}]+)\}/g, (match, expr: string) => {
        const trimmed = expr.trim();
        // Check if expression references only known variables
        const refsKnown = KNOWN_VARS.some(v => trimmed.includes(v));
        if (!refsKnown) {
            const key = `__PLACEHOLDER_${idx++}__`;
            placeholders.set(key, match);
            return key;
        }
        return match;
    });

    // Step 2: Build and call new Function with the 5 known variables
    try {
        const body = `return \`${escaped}\`;`;
        const fn = new Function('description', 'start', 'duration', 'tags', 'project', body);
        let result = fn(vars.description, vars.start, vars.duration, vars.tags, vars.project) as string;

        // Step 3: Restore unknown placeholders to their original ${...} text
        for (const [key, original] of placeholders) {
            result = result.replace(key, original);
        }
        return result;
    } catch {
        // Broken template (SyntaxError) — return raw template string as-is
        return template;
    }
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | `renderTemplate` — `new Function` evaluator | accept | User writes their own template in their own vault settings on their own machine. No remote input. `new Function` receives exactly 5 named parameters. Acceptable risk for local desktop plugin — same trust model as Obsidian's own Templater community plugin. |
| T-10-02 | Elevation of Privilege | `renderTemplate` — access to `globalThis` via template | accept | Template like `${globalThis.process}` could theoretically access globals. Mitigated by local trust model: user is executing code on their own machine, same as any Obsidian plugin or Templater template. No escalation beyond what the user already has. |
| T-10-03 | Denial of Service | `renderTemplate` — infinite loop in template expression | accept | A template like `${(() => { while(true){} })()}` would freeze the plugin. Same risk exists with any Obsidian plugin that evaluates user expressions. Local desktop app — user harms only themselves. |
</threat_model>
