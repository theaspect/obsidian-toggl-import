    - **Test CMD-01: registers Import Toggl Entries command**
      Behavior: After `await plugin.onload()`, `mockAddCommand` was called exactly once with an object whose `id === 'import-toggl-entries'` and `name === 'Import Toggl Entries'` and whose `editorCallback` is a function.
      Assertion: `expect(mockAddCommand).toHaveBeenCalledOnce()`, `expect(spec.id).toBe('import-toggl-entries')`, `expect(spec.name).toBe('Import Toggl Entries')`, `expect(typeof spec.editorCallback).toBe('function')`.

      Behavior: When `getActiveFile()` returns `{ basename: '2024-06-15' }` and apiToken is set, calling `editorCallback(editor)` calls `mockFetchTimeEntries` with `(pluginInstance, '2024-06-15')`.
      Setup: `mockFetchTimeEntries.mockResolvedValue([{ id: 1 }])`, `mockFormatEntries.mockReturnValue('formatted')`.
      Assertion: `expect(mockFetchTimeEntries).toHaveBeenCalledWith(expect.anything(), '2024-06-15')`.

    - **Test CMD-03: rejects non-date filename, shows notice, makes no API call**
      Behavior: When `getActiveFile()` returns `{ basename: 'not-a-date' }`, calling `editorCallback(editor)` shows the notice "Active note filename is not a valid date (expected yyyy-mm-dd)." and does NOT call `mockFetchTimeEntries`.

    - **Test CMD-03b: empty token guard fires before date check (D-04)**

    - **Test CMD-05: API error message surfaces as Notice**

      Also assert non-Error rejection: `mockFetchTimeEntries.mockRejectedValue('weird')` → Notice called with `'Toggl API error: unknown error'`.
