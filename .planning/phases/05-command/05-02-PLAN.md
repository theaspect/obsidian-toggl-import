   ```typescript
   this.addCommand({
       id: 'import-toggl-entries',
       name: 'Import Toggl Entries',
       editorCallback: async (editor: Editor) => {
           // D-04: Empty token guard — fail fast before any network call
           if (this.settings.apiToken === '') {
               new Notice('Configure your Toggl API token in Settings → Toggl Import first.');
               return;
           }

           // D-07: Date validation from active file basename
           const basename = this.app.workspace.getActiveFile()?.basename ?? '';
           if (!/^\d{4}-\d{2}-\d{2}$/.test(basename)) {
               new Notice('Active note filename is not a valid date (expected yyyy-mm-dd).');
               return;
           }

           // D-05: Fetch with structured error handling
           let entries: TimeEntry[];
           try {
               entries = await fetchTimeEntries(this, basename);
           } catch (err: unknown) {
               new Notice(err instanceof Error ? err.message : 'Toggl API error: unknown error');
               return;
           }

           // D-06: Empty result short-circuit (no insert)
           const formatted = formatEntries(entries, this.settings);
           if (formatted === '') {
               new Notice('No entries found for this date.');
               return;
           }

           // D-01 / D-02 / D-03: Insert at cursor with trailing newline + success notice
           editor.replaceSelection(formatted + '\n');
           new Notice(`Imported ${entries.length} entries`);
       },
   });
   ```

3. **Do NOT** add any sentinel comments, dedup logic, or "v1/placeholder" patterns. Insert-only is the locked contract — REIMP-01 is satisfied trivially.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Obsidian editor → plugin | User-controlled note basename crosses into the plugin and is used as a Toggl API parameter |
| Plugin → Toggl API | Authenticated HTTPS call carrying the user's API token (handled in api.ts, not this phase) |
| Plugin → Editor | Plugin writes formatted text into a user note via `replaceSelection` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | basename → API URL | mitigate | Strict regex `/^\d{4}-\d{2}-\d{2}$/` validates basename BEFORE it is passed to `fetchTimeEntries`. Anything else is rejected with a Notice and no API call. (D-07) |
| T-05-05 | Information Disclosure | error Notice | mitigate | Error messages from `api.ts` are pre-formatted and known to never include the API token (verified by api.test.ts). The catch block surfaces only `error.message`, not the full error object or stack trace. |
| T-05-06 | Denial of Service | rapid command spam | accept | Toggl's own rate limiter (429 handled in api.ts) is the chokepoint. The command itself is a single async call per invocation; no internal loop or queue. Low risk for a single-user desktop plugin. |
| T-05-07 | Elevation of Privilege | editorCallback execution | accept | `editorCallback` only fires when an editor is open and focused; Obsidian gates this. No code path runs without user-initiated command palette or shortcut activation. |
| T-05-08 | Tampering | malicious formatter output | accept | The formatter is internal code (Phase 4) producing only Markdown table or delimited text from typed `TimeEntry` data. No untrusted HTML/JS injection surface. |
| T-05-09 | Information Disclosure | empty token leak via 401 | mitigate | D-04 empty-token guard fires BEFORE any network call, so a blank token cannot accidentally leak via a 401 server response logged by an upstream proxy. |

**Block on high severity:** No high-severity threats. All threats are mitigated by locked decisions (D-04, D-07) or accepted with rationale. Plan is safe to execute.
</threat_model>
